# Domain vocabulary

Definitions for terms that recur across the codebase, docs, and reviews. Use these exactly — don't drift to "tenant," "client repo," "service," "endpoint," or other near-synonyms.

## Domain terms

**Hub Repo**
This repo (`Fralleee/agent-workflows`). Holds the **Reusable Workflow** and the agent **Prompt**. The thing every consumer's stub points back at via `@v1`.

**Consumer Repo**
Any repo the App is installed on. Receives a **Stub Workflow** via the install PR; runs the agent on its cron.

**Reusable Workflow**
[`.github/workflows/daily-scan.yml`](.github/workflows/daily-scan.yml). The actual workflow that runs the agent. Consumer stubs delegate to it via `uses: Hub/.../daily-scan.yml@v1`.

**Stub Workflow**
The minimal workflow the App writes into the Consumer Repo's `.github/workflows/daily-scan.yml`. Just declares the schedule, permissions, secrets passthrough, and the chosen Agent Profile's `provider:` + `model:`. Generated by `installer/src/stub.ts`.

**Prompt**
[`prompts/daily-scan.md`](prompts/daily-scan.md). The decision tree the agent runs on every cron tick. Lives in the Hub; pulled fresh from `main` on each Reusable Workflow run.

**Agent Profile**
A curated `(provider, model, secret-name)` triple users pick from in the Setup Form. Catalog lives in `installer/src/profiles.ts`. The "Custom" entry lets advanced users specify their own provider+model directly.

**Install Flow**
The end-to-end user journey: GitHub install screen → Setup Form → POST /install → per-repo (set secret, ensure label, open PR).

**Setup Form**
The HTML form at `GET /setup?installation_id=N`. Where users pick an Agent Profile and paste their API key. Rendered by the installer Worker.

**Install Token**
A GitHub-issued, installation-scoped REST token. Minted on demand by exchanging a JWT signed with the App's RSA private key. Used for every per-Consumer-Repo API call during install.

**Installer (Worker)**
The Vercel Edge function under [`installer/`](installer/). Renders the Setup Form, orchestrates the Install Flow, verifies webhooks. Stateless — no DB, no persisted secrets.
